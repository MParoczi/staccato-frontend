## Critical Anti-Patterns (do NOT repeat these)

- **Don't replace ModuleCard wholesale** ‚Äî its F8 contracts (`data-testid={module-card-${id}}`, `data-selected`, `data-conflicting`, `data-dragging`, `aria-pressed={isSelected}`, header drag listeners gated by `isSelected`, ModuleResizeHandles render gate) are load-bearing. Surgical insertion only: add edit-mode state + Suspense boundary + click-outside + EditButton render branch.
- **Don't write a `react-i18next` test mock that returns translated strings** ‚Äî the precedent across all Phase 1 tests is `t: (k) => k` so assertions match raw keys. Diverging breaks future test reuse.
- **Don't run `pnpm run build` as a gate** ‚Äî pre-existing TS errors in `profile/useLanguageSwitch.test.tsx`, `styling/StyleEditorDrawer.test.tsx`, `routes/notebook-layout.test.tsx` predate Phase 1 (documented in plan 01-01 SUMMARY). Use `pnpm tsc --noEmit` + `pnpm test --run` as the real verification gate. The Vite chunk-split check from task 6.4's acceptance criteria is the ONE place build matters; if build still fails, document and use `pnpm vite build` directly (skips `tsc -b`).
