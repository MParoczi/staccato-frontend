<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public/locales/ → browser | Translation JSON files are served as static assets; untrusted if a CDN or static host is compromised |
| i18next language detector → browser | Language detector reads navigator.language; attacker controlling browser locale could change detected language |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-10 | Tampering | Translation file injection via CDN compromise | accept | Translation files contain UI strings only (no code); no eval() or innerHTML usage in i18next; XSS via translation strings requires server compromise which is out of scope for ASVS L1 |
| T-01-11 | Information Disclosure | Language detection reveals browser locale | accept | i18next.language is only 'en' or 'hu' (supportedLngs constraint); no PII is leaked; low risk |
| T-01-12 | Tampering | Invalid JSON in translation files causes runtime error | mitigate | JSON syntax validated in acceptance criteria; malformed files cause i18next to fall back to fallbackLng 'en'; app degrades gracefully |
| T-01-SC | Tampering | npm/pnpm installs | mitigate | i18next@26.2.0, react-i18next@17.0.8, i18next-http-backend@4.0.0 verified as Approved in Package Legitimacy Audit (RESEARCH.md) |
</threat_model>
