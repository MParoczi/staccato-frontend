<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ProtectedRoute → /app/* routes | Unauthenticated users must not reach /app/* routes; ProtectedRoute enforces this |
| / (RootPage) → navigation targets | Redirect destinations are hardcoded (/login, /app/notebooks) — no user-controlled redirect input |
| main.tsx boot → /auth/refresh | rawClient sends refresh cookie; backend validates and returns new token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-13 | Elevation of Privilege | ProtectedRoute bypass | mitigate | ProtectedRoute gates all /app/* routes; status 'loading' shows spinner (no bypass); status 'unauthenticated' returns Navigate (ASVS V4.1.1 — enforce access control server side; client-side gating adds UX defense-in-depth) |
| T-01-14 | Spoofing | Open redirect | mitigate | Navigate targets are hardcoded string literals (/login, /app/notebooks); no user-supplied redirect URL is accepted; window.location.href forbidden (CLAUDE.md) |
| T-01-15 | Tampering | Silent refresh response — malformed JSON | accept | rawClient.post response is typed; TypeScript provides compile-time check; malformed JSON causes .catch() → clearAuth() → redirect to /login; no data corruption |
| T-01-16 | Denial of Service | i18n Suspense blocking render indefinitely | accept | i18next-http-backend loads from same-origin /locales/ (served by Vite or CDN); if files fail to load, fallbackLng='en' is used; i18next degrades gracefully |
| T-01-SC | Tampering | npm/pnpm installs | mitigate | react-router@7.15.1, @tanstack/react-query@5.100.10 verified as Approved in Package Legitimacy Audit (RESEARCH.md) |
