    Request interceptor (client.interceptors.request.use):
    - Read token: const token = useAuthStore.getState().accessToken
    - If token is non-null: set config.headers.Authorization = 'Bearer ' + token
    - Always set config.headers['Accept-Language'] = i18next.language ?? 'en'
    - Return config

    Response interceptor (client.interceptors.response.use):
    - Pass-through on success: (response) => response
    - Error handler:
      - Declare module-level let refreshPromise: Promise<string> | null = null (outside the interceptor setup, inside the module)
      - If error.response?.status !== 401 OR originalRequest._retried is true: reject with error
      - Otherwise: set originalRequest._retried = true
      - If refreshPromise is null: assign refreshPromise = rawClient.post('/auth/refresh').then(res => { const token = res.data.accessToken; useAuthStore.getState().setAuth(useAuthStore.getState().user!, token); return token; }).catch(err => { useAuthStore.getState().clearAuth(); throw err; }).finally(() => { refreshPromise = null; })
      - Await refreshPromise to get newToken
      - Set originalRequest.headers.Authorization = 'Bearer ' + newToken
      - Retry: return client(originalRequest)

    Export client as named export.

    IMPORTANT: The rawClient.post call should type its response as { accessToken: string; user: UserProfile } — use generic: rawClient.post<{ accessToken: string; user: UserProfile }>('/auth/refresh').

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Zustand store → DOM | accessToken lives in JS memory; never written to DOM, localStorage, or sessionStorage |
| client.ts → API server | Every request crosses this boundary; auth token attached via Bearer header |
| rawClient.ts → /auth/refresh | Refresh token in HttpOnly cookie; JS cannot read it, only browsers send it automatically |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Information Disclosure | accessToken in authStore | mitigate | Token in Zustand memory only; no persist middleware; not in DOM, not in localStorage; cleared by clearAuth() on refresh failure (ASVS V2.1.1) |
| T-01-07 | Denial of Service | Concurrent 401s (token stampede) | mitigate | module-level refreshPromise ensures all concurrent 401s share one refresh call; only one POST /auth/refresh per burst (PROJECT.md Key Decisions: "Single-flight refresh pattern") |
| T-01-08 | Elevation of Privilege | Circular refresh loop | mitigate | rawClient (no interceptors) used for /auth/refresh endpoint; using client would cause 401 → refresh → 401 infinite loop; rawClient breaks the cycle (PROJECT.md Key Decisions: "rawClient for auth/refresh") |
| T-01-09 | Spoofing | Accept-Language header injection | accept | Header value sourced from i18next.language (controlled by app); only 'en' or 'hu' are supported values from the language detector; low risk |
| T-01-SC | Tampering | npm/pnpm installs | mitigate | zustand@5.0.13 and axios@1.16.1 verified as Approved in Package Legitimacy Audit (RESEARCH.md) |
</threat_model>
