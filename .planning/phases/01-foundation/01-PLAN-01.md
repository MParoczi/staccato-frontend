<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build environment → Vite | `import.meta.env` values injected at build time from `.env` files; untrusted if .env is editable by an attacker |
| Browser → API | All HTTP requests from `src/api/client.ts` cross this boundary; credentials in HttpOnly cookie |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `import.meta.env.VITE_API_BASE_URL` | mitigate | Validated as a URL format by Zod in `src/env.ts` at startup; app throws before rendering if invalid (ASVS V5.1.1 — reject untrusted data) |
| T-01-02 | Information Disclosure | Auth token storage | mitigate | Access token stored only in Zustand in-memory state; no localStorage, no sessionStorage, no persist middleware; not reachable via script injection (ASVS V2.1.1, V3.3.1) |
| T-01-03 | Spoofing | Navigation / redirects | mitigate | All navigation via React Router `<Navigate>` or `useNavigate`; `window.location.href` forbidden; open redirect via crafted URLs is not exploitable |
| T-01-04 | Tampering | npm install supply chain | mitigate | All packages verified in Package Legitimacy Audit (RESEARCH.md); all flagged as Approved with known authoritative GitHub repos; no postinstall scripts |
| T-01-05 | Information Disclosure | `src/env.ts` error messages | accept | Error messages logged to console include env var names but not values; acceptable in dev environment; VITE_ vars are not secrets (they are bundled into the JS artifact anyway) |
| T-01-SC | Tampering | npm/pnpm installs | mitigate | Package Legitimacy Audit completed in RESEARCH.md; all packages Approved; no [ASSUMED] or [SUS] packages require a blocking checkpoint |
</threat_model>
