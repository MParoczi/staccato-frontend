Error extraction helper (inline in this file):
```typescript
function extractErrorMessage(error: unknown, fallback: string): string {
  if (error && typeof error === 'object' && 'response' in error) {
    const resp = (error as { response?: { data?: { detail?: string } } }).response
    if (resp?.data?.detail) return resp.data.detail
  }
  return fallback
}
```

Component: `export function DeleteNotebookDialog({ notebook, open, onOpenChange }: DeleteNotebookDialogProps)`:

1. Early return: `if (!notebook) return null` — guard against rendering with no target.
2. `const { t } = useTranslation('notebooks')`
3. `const queryClient = useQueryClient()`
4. Mutation:
```typescript
const mutation = useMutation({
  mutationFn: () => deleteNotebook(notebook.id),
  onSuccess: () => {
    queryClient.invalidateQueries({ queryKey: ['notebooks'] })
    onOpenChange(false)
  },
  onError: (error) => {
    toast.error(extractErrorMessage(error, t('errors.deleteFailed')))
  },
})
```

5. Return Dialog:
```tsx
<Dialog open={open} onOpenChange={onOpenChange}>
  <DialogContent>
    <DialogHeader>
      <DialogTitle>{t('delete.title', { name: notebook.title })}</DialogTitle>
      <DialogDescription>{t('delete.description')}</DialogDescription>
    </DialogHeader>
    <DialogFooter>
      <Button
        variant="outline"
        onClick={() => onOpenChange(false)}
        disabled={mutation.isPending}
      >
        {t('cancel', { ns: 'common' })}
      </Button>
      <Button
        variant="destructive"
        onClick={() => mutation.mutate()}
        disabled={mutation.isPending}
      >
        {mutation.isPending && <Loader2 className="mr-2 size-4 animate-spin" />}
        {t('delete.confirm')}
      </Button>
    </DialogFooter>
  </DialogContent>
</Dialog>
```

Note on `t('cancel', { ns: 'common' })`: this accesses the 'common' namespace Cancel label from within a component that primarily uses 'notebooks' namespace. This is a valid i18next pattern.

| T-04-13 | XSS | Notebook title in delete dialog title (via i18next interpolation) | mitigated | i18next escapes interpolated values by default; `t('delete.title', { name: notebook.title })` does not inject raw HTML |
| T-04-14 | Tampering | DELETE /notebooks/:id | accept | Backend validates JWT ownership; unauthorized deletes return 403; frontend shows error toast |
| T-04-15 | Denial of Service | Double-click delete button | mitigated | Button is disabled while mutation.isPending; only one DELETE request fires per dialog open |
| T-04-16 | Information Disclosure | Delete error toast showing API detail | accept | User sees their own notebook error messages only; no PII beyond their own data |
| T-04-SC | Tampering | pnpm installs | accept | No new packages added in this plan |
