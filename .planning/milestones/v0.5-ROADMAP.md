## Milestone Summary

**Key Decisions:**
- Lessons feature scoped to `src/features/lessons/` — same feature-scoped structure as notebooks
- LessonPage uses URL search params (`?page=N`) for page navigation — page position survives refresh and back/forward
- LessonsPage is a child of `notebooks/:id` (inherits tab bar chrome); LessonPage is a sibling at AppLayout level (full-screen, no tab bar)
- `globalPageStart` is 0-indexed from backend — frontend adds +1 for display
- `POST /lessons/{id}/pages` response does not include `pageNumber` — frontend computes `totalPages + 1` client-side
- `extractErrorMessage` defined inline per feature file — respects cross-feature import boundary (CLAUDE.md constraint)
- All HU translation strings are `__HU_TODO__` stubs — full HU translation deferred to Phase 12

**Bugs found and fixed during UAT:**
1. T-04: "Notebook p. 0" — `globalPageStart` is 0-indexed; added +1 to computed value in `LessonPage.tsx`
2. T-04: Dotted-grid canvas not rendering — `hsl(var(--muted-foreground) / 0.25)` invalid in Tailwind v4 oklch; switched to `color-mix(in oklch, ...)`
3. T-07: NaN in URL after adding page — `newPage.pageNumber` undefined (backend field mismatch); switched to `totalPages + 1` client-side
4. T-05: Old title showed in LessonPage after rename — added `['lesson', lessonId]` cache invalidation to `RenameLessonDialog`

**Backend Discoveries:**
- `globalPageStart` is 0-indexed (value 0 = first lesson, no preceding pages in the notebook)
- `POST /lessons/{id}/pages` response does not include a `pageNumber` field matching the frontend's `LessonPage` type
