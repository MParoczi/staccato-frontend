## Milestone Summary

**Key Decisions:**

- `COVER_COLORS`, `NOTEBOOK_STYLE_PRESETS`, `NOTEBOOK_PAGE_SIZES` exported as `as const` arrays — no enum keyword (erasableSyntaxOnly constraint)
- `PageErrorBoundary` implemented as a class component — only React pattern that supports `getDerivedStateFromError`
- `NotebookFormDialog` in dual create/edit mode — instrument field hidden in edit mode (backend rejects immutable field changes)
- `extractErrorMessage` inlined per component (not a shared utility) — prevents global double-firing of error toasts
- Navbar breadcrumb uses `staleTime: 300_000` (5 min) — avoids redundant `GET /notebooks/:id` calls on navigation
- Instrument hardcoded as "Guitar" in all Phase 4 UI — backend only supports 6-string guitar; generalize in a future phase

**Issues Resolved:**

- React hooks-before-return ordering in `DeleteNotebookDialog` — hooks placed before early return to satisfy rules of hooks
- `PUT /notebooks/:id` rejects immutable fields — edit payload excludes `pageSize` and `instrumentId` (mutable: title + coverColor only)

**Issues Deferred:**

- T-09 (offline mutation error toast): DevTools offline stalls Axios before the 15s timeout fires; 4xx errors do toast correctly. Accepted gap documented in `04-UAT.md`.
- Hungarian notebook strings are `__HU_TODO__` stubs — full HU translation deferred to Phase 12

**Technical Debt Incurred:**

- `GET /instruments` assumed shape matches `InstrumentOption[]`; update interface if endpoint changes
- No unit tests added for Phase 4 notebook components — integration via UAT only
