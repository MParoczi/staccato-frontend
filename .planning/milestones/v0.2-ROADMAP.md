## Overview

Users can register, log in via email/password or Google OAuth, stay logged in across page reloads via HttpOnly refresh cookie, and log out cleanly. The app silently restores session state on every page load via `POST /auth/refresh`. The proactive refresh hook renews tokens 60 seconds before expiry.

All 5 success criteria verified by UAT (10/10 tests passed) and automated verification (5/5 must-haves, 25/25 tests, TypeScript clean).

## Milestone Summary

**Key Decisions:**

- @hookform/resolvers pinned to 3.10.0 (v5.2.2 imports zod/v4/core, incompatible with zod 3.24.4)
- navigate('/login', { replace: true }) in logout — prevents back-button re-entry into protected route
- useProactiveRefresh mounted unconditionally in ProtectedRoute before status checks (React Rules of Hooks)
- Zod schema wrapped in useMemo + t — i18n-reactive validation messages without re-creating schema on every render
