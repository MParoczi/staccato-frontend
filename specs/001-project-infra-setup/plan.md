## Summary

Set up the foundational infrastructure for the Staccato frontend SPA — a React TypeScript app on Vite. This feature installs all required dependencies, configures Tailwind CSS v4 with a custom earthy design system, initializes shadcn/ui with overridden theme colors, sets up Axios with 401 silent-refresh interceptors, creates Zustand stores, configures TanStack Query, defines the React Router v7 route structure with protected routes, initializes react-i18next, and provides all shared TypeScript types, enums, and constants mirroring the backend API contract. No actual pages or UI — just the wiring that all subsequent features depend on.

## Design Decisions

### D1. Axios Interceptor Chain

Two Axios instances prevent infinite refresh loops structurally:

```
api (main instance)                    rawApi (bare instance)
  ├── Defaults:                          ├── baseURL from env
  │   ├── timeout: 30000 (30s)          ├── withCredentials: true
  │   └── withCredentials: true          └── (no interceptors)
  ├── Request interceptor:
  │   ├── Authorization header
  │   └── Accept-Language header
  └── Response interceptor:
      └── 401 → refresh via rawApi
          ├── Success → retry original
          └── Failure → clearAuth, redirect /login
```

Concurrent 401 dedup uses a shared promise (module-level `refreshPromise` variable). All pending 401 requests await the same promise.

### D2. ProtectedRoute Flow

```
ProtectedRoute mount
  ├── Token in store? → render children
  └── No token?
      ├── Save current URL as intended destination
      ├── Show loading spinner
      ├── Call silentRefresh() via api/auth.ts
      ├── Success? → token set, re-render, children visible
      └── Failure? → redirect to /login (with intended URL preserved)

Post-login redirect: after successful login, redirect to saved intended URL
(default: /app/notebooks if no intended URL saved)
```

The `silentRefresh()` function in `api/auth.ts` reuses the same shared-promise dedup from D1 (exported from `api/client.ts`), preventing double-refresh from nested route guards.

### D3. Tailwind v4 Theme Architecture

```css
/* Layer 1: Raw values in :root / .dark */
:root {
  --primary: oklch(0.50 0.10 55);         /* warm brown */
  --background: oklch(0.97 0.008 75);     /* cream */
  /* ... */
}
.dark {
  --primary: oklch(0.70 0.08 55);         /* lighter brown for dark bg */
  --background: oklch(0.18 0.02 55);      /* deep warm brown */
  /* ... */
}

/* Layer 2: Wire into Tailwind via @theme inline */
@theme inline {
  --color-primary: var(--primary);
  --color-background: var(--background);
  /* ... */
}
```

All shadcn/ui components use Tailwind utilities (`bg-primary`, `text-foreground`) which resolve through this chain automatically.

### D4. TanStack Query Configuration

```
QueryClient defaults:
  staleTime: 0              (constitution: notebooks/lessons/modules always refetch)
  gcTime: 300_000           (5 min garbage collection)
  retry: (count, error) =>
    error.status === 401 ? false : count < 3
  refetchOnWindowFocus: true

Per-query overrides (applied in feature hooks):
  chords/instruments: staleTime: 300_000
  user profile: staleTime: 30_000
```

### D5. i18n Configuration

```
i18next init:
  fallbackLng: 'en'
  detection: navigator.language → 'en' or 'hu'
  resources: en.json, hu.json (loaded eagerly, small files)
  interpolation: { escapeValue: false } (React handles escaping)
  ns: default namespace
  keySeparator: '.'

Post-login: user.language overrides detected language
Language change updates: i18next locale + Axios Accept-Language header
```

Translation key namespaces (per Constitution Principle IX):
- `auth.*` — login, register, password reset
- `notebooks.*` — dashboard, create, cover
- `editor.*` — blocks, modules, canvas
- `chords.*` — library, search, detail
- `exports.*` — export history, status
- `profile.*` — settings, deletion
- `common.*` — shared strings (save, cancel, delete, error messages)
